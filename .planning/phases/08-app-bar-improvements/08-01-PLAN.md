<task type="auto">
  <name>Task 1: Fix app-bar.css — dead code, tokens, touch targets, motion, responsive</name>
  <files>packages/ui/src/components/app-bar/app-bar.css</files>
  <action>
Edit `app-bar.css` with these specific changes:

1. **Remove dead CSS (APPBAR-01):** Delete the `.app-bar__mobile-nav` rule block (lines 148-156) — this class is never rendered by the component. The component uses `.app-bar__nav--mobile-open` instead.

2. **Fix inconsistent tokens (APPBAR-02):** Replace `--g-spacing-12` with `--g-theme-spacing-sm` in:
   - `.app-bar__actions` gap property (line 77)
   - `::slotted(a)` padding-inline property (line 204)

3. **Fix sticky behavior (APPBAR-03):** In the `.app-bar` base rule, change `position: sticky` and `inset-block-start: 0` to `position: relative`. Move `position: sticky` and `inset-block-start: 0` into `.app-bar--sticky` (which currently only exists implicitly via classMap but has no styles).

   Add new rule:
   ```css
   .app-bar--sticky {
     position: sticky;
     inset-block-start: 0;
   }
   ```

4. **Touch target minimum (APPBAR-04):** Change `.app-bar__menu-button` from `inline-size: 2.5rem; block-size: 2.5rem;` to `inline-size: 2.75rem; block-size: 2.75rem;` (44px).

5. **Reduced motion (APPBAR-05):** Add at the end (before slotted content styles):
   ```css
   @media (prefers-reduced-motion: reduce) {
     .app-bar__menu-icon::before,
     .app-bar__menu-icon::after {
       transition: none;
     }

     .app-bar__menu-button {
       transition: none;
     }

     @keyframes slide-down {
       from { opacity: 1; transform: none; }
     }
   }
   ```

6. **Fix transition: all (APPBAR-06):** In `::slotted(a)`, change `transition: all 0.2s ease;` to `transition: color 0.2s ease, background-color 0.2s ease;`

7. **Mobile ul stacking (APPBAR-07):** Add a rule inside the mobile-open context. Since `::slotted` can't be combined with descendant selectors in shadow DOM, add to the `.app-bar__nav--mobile-open` rule:
   ```css
   .app-bar__nav--mobile-open ::slotted(ul) {
     flex-direction: column;
   }
   ```
   Note: `::slotted(ul)` already has `display: flex` and `flex-direction: row`. The mobile-open override will change direction. This works because `.app-bar__nav--mobile-open ::slotted(ul)` has higher specificity than `::slotted(ul)`.

8. **Clean up duplicate container query blocks:** There are two `@container app-bar (min-width: 768px)` blocks (lines 67-71 and 159-186). Consolidate them into a single block. The second block already contains the `.app-bar__nav` desktop styles — merge the first block's `.app-bar__nav { display: flex; }` into the second block's `.app-bar__nav` rule, then delete the first block.

Also consolidate the two `.app-bar__menu-button { display: none; }` rules (lines 101-105 is a separate container query block — merge into the main one).
