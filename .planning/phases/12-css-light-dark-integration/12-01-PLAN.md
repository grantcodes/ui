  <action>
For each of the 8 tier-2 JSON files (4 per theme: grantcodes and wireframe), merge the dark override values into the light file using `light-dark()` syntax.

**Merge algorithm per token:**
1. Read the light value and corresponding dark value for the same JSON path
2. If the light value and dark value are DIFFERENT: set `"value": "light-dark({lightRef}, {darkRef})"` — keeping the original `{reference}` syntax inside the expression
3. If the light value and dark value are IDENTICAL: keep the plain value (no `light-dark()` wrapper needed)
4. If a token exists in light but NOT in dark: keep the plain light value
5. If a token exists in dark but NOT in light: this shouldn't happen for tier-2, but if it does, add it with `light-dark()` wrapping

**Example transformation for grantcodes background.json:**

Light `"default"` is `"{color.neutral.white}"`, dark `"default"` is `"{color.neutral.900}"`
→ Result: `"default": { "value": "light-dark({color.neutral.white}, {color.neutral.900})" }`

Light `"transparent"` is `"{color.transparent.25}"`, dark `"transparent"` is `"{color.transparent.25}"`
→ Result: `"transparent": { "value": "{color.transparent.25}" }` (unchanged — same value)

**Shadow token handling:**
For shadows.json, the structure has sub-keys (x, y, blur, spread, color). Only the `color` sub-key differs between light and dark. The positional properties (x, y, blur, spread) reference the same tier-1 shadow values in both modes and should NOT get `light-dark()`. Only wrap `color` if it differs.

For grantcodes shadows.json:
- Light `sm.color` = `"{shadow.sm.color}"`, dark `sm.color` = `"#00000080"` → `"color": { "value": "light-dark({shadow.sm.color}, #00000080)" }`
- Light `sm.x` = `"{shadow.sm.x}"`, dark `sm.x` = `"{shadow.sm.x}"` → keep as `"{shadow.sm.x}"` (unchanged)

**Wireframe notes:**
Wireframe has additional tokens not present in grantcodes (e.g. `dataviz` sub-object in background). Apply the same merge algorithm — compare each path between light and dark files. Wireframe dark also has `dataviz` entries so merge them too.

Preserve existing JSON formatting: tab indentation, structure, ordering.
